title: Event Types
description: Event types emitted during Codebuff agent execution
---

The `handleEvent` callback receives events throughout the agent's execution lifecycle. Use these events to track progress, display UI updates, or log agent activity.

## Event Types

All events are discriminated by the `type` field.

```typescript
type PrintModeEvent =
  | PrintModeStart
  | PrintModeError
  | PrintModeDownloadStatus
  | PrintModeToolCall
  | PrintModeToolResult
  | PrintModeText
  | PrintModeFinish
  | PrintModeSubagentStart
  | PrintModeSubagentFinish
  | PrintModeReasoningDelta
```

---

## Start Event

Emitted when the agent begins execution.

<ResponseField name="type" type="'start'" required>
  Event type identifier
</ResponseField>

<ResponseField name="agentId" type="string">
  Unique identifier for the agent instance
</ResponseField>

<ResponseField name="messageHistoryLength" type="number" required>
  Number of messages in the conversation history
</ResponseField>

**Example:**
```typescript
{
  type: 'start',
  agentId: 'agent-abc123',
  messageHistoryLength: 5
}
```

---

## Error Event

Emitted when an error occurs during execution.

<ResponseField name="type" type="'error'" required>
  Event type identifier
</ResponseField>

<ResponseField name="message" type="string" required>
  Error message describing what went wrong
</ResponseField>

**Example:**
```typescript
{
  type: 'error',
  message: 'Failed to connect to API: Network timeout'
}
```

## Download Status Event

Emitted during model or dependency downloads.

<ResponseField name="type" type="'download'" required>
  Event type identifier
</ResponseField>

<ResponseField name="version" type="string" required>
  Version being downloaded
</ResponseField>

<ResponseField name="status" type="'complete' | 'failed'" required>
  Download completion status
</ResponseField>

**Example:**
```typescript
{
  type: 'download',
  version: '1.2.0',
  status: 'complete'
}
```

---

## Tool Call Event

Emitted when the agent calls a tool.

<ResponseField name="type" type="'tool_call'" required>
  Event type identifier
</ResponseField>

<ResponseField name="toolCallId" type="string" required>
  Unique identifier for this tool call
</ResponseField>

<ResponseField name="toolName" type="string" required>
  Name of the tool being called (e.g., `write_file`, `run_terminal_command`)
</ResponseField>

<ResponseField name="input" type="Record<string, any>" required>
  Input parameters passed to the tool
</ResponseField>

<ResponseField name="agentId" type="string">
  ID of the agent making the call
</ResponseField>

<ResponseField name="parentAgentId" type="string">
  ID of the parent agent (if this is a subagent)
</ResponseField>

<ResponseField name="includeToolCall" type="boolean">
  Whether to include this tool call in the message history
</ResponseField>

**Example:**
```typescript
{
  type: 'tool_call',
  toolCallId: 'call_123',
  toolName: 'write_file',
  input: {
    path: 'src/index.ts',
    content: 'console.log("Hello")'
  },
  agentId: 'agent-abc123'
}
```

---

## Tool Result Event

Emitted when a tool call completes.

<ResponseField name="type" type="'tool_result'" required>
  Event type identifier
</ResponseField>

<ResponseField name="toolCallId" type="string" required>
  ID matching the original tool call
</ResponseField>

<ResponseField name="toolName" type="string" required>
  Name of the tool that executed
</ResponseField>

<ResponseField name="output" type="ToolResultOutput[]" required>
  Array of output results from the tool execution
</ResponseField>

<ResponseField name="parentAgentId" type="string">
  ID of the parent agent (if this is a subagent)
</ResponseField>

**Example:**
```typescript
{
  type: 'tool_result',
  toolCallId: 'call_123',
  toolName: 'write_file',
  output: [
    {
      type: 'json',
      value: { success: true, path: 'src/index.ts' }
    }
  ]
}
```

## Subagent Start Event

Emitted when a subagent begins execution.

<ResponseField name="type" type="'subagent_start'" required>
  Event type identifier
</ResponseField>

<ResponseField name="agentId" type="string" required>
  Unique identifier for the subagent instance
</ResponseField>

<ResponseField name="agentType" type="string" required>
  Type of the subagent (e.g., `file_picker`, `researcher`)
</ResponseField>

<ResponseField name="displayName" type="string" required>
  Human-readable name for the subagent
</ResponseField>

<ResponseField name="onlyChild" type="boolean" required>
  Whether this is the only subagent in the current step
</ResponseField>

<ResponseField name="parentAgentId" type="string">
  ID of the parent agent
</ResponseField>

<ResponseField name="params" type="Record<string, any>">
  Parameters passed to the subagent
</ResponseField>

<ResponseField name="prompt" type="string">
  Prompt given to the subagent
</ResponseField>

**Example:**
```typescript
{
  type: 'subagent_start',
  agentId: 'subagent-xyz',
  agentType: 'file_picker',
  displayName: 'File Picker',
  onlyChild: true,
  parentAgentId: 'agent-abc123',
  prompt: 'Find relevant test files'
}
```

## Reasoning Delta Event

Emitted during the agent's reasoning process (streaming chunks).

<ResponseField name="type" type="'reasoning_delta'" required>
  Event type identifier
</ResponseField>

<ResponseField name="text" type="string" required>
  Chunk of reasoning text
</ResponseField>

<ResponseField name="ancestorRunIds" type="string[]" required>
  Array of ancestor agent run IDs
</ResponseField>

<ResponseField name="runId" type="string" required>
  ID of the current agent run
</ResponseField>

**Example:**
```typescript
{
  type: 'reasoning_delta',
  text: 'I need to check the file first...',
  ancestorRunIds: ['run-parent'],
  runId: 'run-abc123'
}
```

## Usage Example

```typescript
import { CodebuffClient } from '@codebuff/sdk'

const client = new CodebuffClient({
  apiKey: 'cb_xxx',
  handleEvent: (event) => {
    switch (event.type) {
      case 'start':
        console.log('Agent started with', event.messageHistoryLength, 'messages')
        break
      
      case 'tool_call':
        console.log('Calling tool:', event.toolName, 'with input:', event.input)
        break
      
      case 'tool_result':
        console.log('Tool result for', event.toolName, ':', event.output)
        break
      
      case 'text':
        process.stdout.write(event.text)
        break
      
      case 'error':
        console.error('Error:', event.message)
        break
      
      case 'finish':
        console.log('Agent finished. Cost:', event.totalCost)
        break
    }
  }
})

const result = await client.run({
  agent: 'base',
  prompt: 'Create a hello world script'
})
```

## Tool Result Output

The `output` field in tool result events contains an array of output objects:

<ResponseField name="type" type="'json' | 'media'" required>
  Type of the output content
</ResponseField>

### JSON Output

<ResponseField name="value" type="any" required>
  JSON value returned by the tool
</ResponseField>

**Example:**
```typescript
{
  type: 'json',
  value: { success: true, filesWritten: 3 }
}
```

### Media Output

<ResponseField name="data" type="string" required>
  Base64-encoded media data
</ResponseField>

<ResponseField name="mediaType" type="string" required>
  MIME type of the media (e.g., `image/png`, `application/pdf`)
</ResponseField>

**Example:**
```typescript
{
  type: 'media',
  data: 'iVBORw0KGgoAAAANSUhEUgA...',
  mediaType: 'image/png'
}
```
